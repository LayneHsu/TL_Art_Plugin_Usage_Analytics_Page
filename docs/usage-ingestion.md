# Usage ingestion service

The ingestion surface is a server-only Firebase Functions boundary. It accepts a POST body with exactly `queue_binding_id`, `lease_token`, and `events`. The caller must present a valid plugin lease issued by the plugin-auth domain. `PluginAuthDecisionService.authorizeEvent` verifies the signature and expiry, checks that the queue binding matches the lease, and re-reads the current binding and principal, so a binding revocation or principal disable takes effect immediately. Portal Firebase tokens, portal UIDs, roles, names, avatars, and client-supplied principal fields are never accepted as event identity.

The checked-in registry is `draft`, so the production function is deliberately unavailable until Task 7 publishes a reviewed non-empty active registry. The runtime reads only the registry, schemas, shared semantic validator, and generated hash/version manifest bundled into the Functions artifact. It does not accept registry JSON or a matching hash from runtime parameters or requests. Startup verifies all artifact hashes and versions, then enforces the same nonempty, stable-key uniqueness, and lifecycle rules as `scripts/validate-contracts.mjs`. Unit and emulator fixtures may inject a custom registry into `UsageIngestionService` and set `allowDraftForTests` explicitly; neither option is exposed by the production endpoint.

Each event is validated independently. A permanent schema, version, registry, tool/action, identity, time-range, or sensitive-field failure is written as bounded `deadLetters` metadata and returned as `permanent_rejected`. Storage failures and rate limits are `retryable`; the result carries `exponential_jitter_until_lease_expiry` so the client can retain those queue entries and retry without exceeding the one-hour lease. A duplicate `event_id` is `confirmed` and does not update any aggregate a second time. The response preserves input order and exposes counts plus a lease-renewal hint.

Accepted events are written in one Firestore transaction with the source event, `toolUsageDaily`, `principalUsageDaily`, and (when applicable) a redacted `errorAggregates` record. Only `run_started` increments usage. Daily buckets use corrected client time in `Asia/Shanghai`; terminal duration is bounded to seven days. Aggregates are derived from server-owned event data and clients cannot submit increments.

If a terminal event arrives before its matching start, the transaction stores the bounded event in `usageOperations.pending_terminal` and creates `usageEventReservations/{event_id}` instead of writing `usageEvents` or aggregates. Exact retries are confirmed without consuming quota again, while any different payload attempting to reuse that ID is permanently rejected. When the matching start arrives, one transaction verifies the reservation, writes both source events and aggregates exactly once, clears `pending_terminal`, and deletes the reservation. Replay never creates, consumes, or deletes these live-ingestion reservations.

The server normalizes summaries before storage and computes the v2 SHA-256 error fingerprint from category, normalized summary, and bounded symbolic `call_site`. It requires the producer-supplied fingerprint to match that value exactly. Paths, Unreal package paths, asset identifiers, emails, credentials, authorization material, cookies, object addresses, tracebacks, and arbitrary diagnostic fields are never persisted as raw values. Error documents use the fingerprint as a grouping value, not the raw summary, retain at most three bounded normalized summary variants, and are split by the schema-bounded `plugin_version`. `affected_versions` therefore remains a one-element compatibility projection instead of an unbounded cross-version array.

The Firestore adapter writes one incoming source event per transaction, or the matching start plus its previously reserved terminal event during terminal-first reconciliation, together with the corresponding aggregate documents. The in-memory adapter used by tests snapshots and restores all maps, including event reservations, on failure to preserve the same atomicity contract. The endpoint adapter is dependency-injected so production wiring cannot silently create a second authentication domain.
