# 数据保留与清理

Spark 方案不使用 Cloud Scheduler、TTL 或 Functions 自动清理。事件和错误日志默认持续保留。管理员在网页的数据管理页手动导出并触发受控批次删除，Rules 只允许启用的管理员删除统计集合。

未来删除范围只包含 `usageDaily` 和 `errorLogs`，按公司日期截止值筛选。网页先显示预计文档数和不可恢复提示，再由管理员确认；删除按小批次执行并显示剩余数量。失败批次保留可重试状态，不能把未删除数据标记为已清理。

删除操作使用 Firebase 当前登录的管理员权限，遵守 Spark 每日删除额度。网页先显示截止日期和最多 1000 条预览，单次最多删除 200 个文档；仍有数据时显示可继续清理状态。建议在删除前下载 JSON/CSV 归档，并在项目外按公司保留制度保存。`pluginUsers` 和 `portalMembers` 不由清理操作删除，成员禁用和资料更新通过网页显式操作。

错误日志调用栈不超过 8 KiB UTF-8，敏感内容在插件写入前脱敏。脱敏规则和合同测试是发布门禁；不能以“已过期”作为跳过脱敏的理由。
