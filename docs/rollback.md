# 回滚策略

## 网页

GitHub Pages 回滚只重新发布已验证的静态构建产物。回滚前确认网页仍使用当前 `usageDaily`、`errorLogs` 和 `portalMembers` 合同；不通过网页回滚删除或修改统计数据。

## Rules 与索引

Firestore Rules 和 Indexes 回滚必须与当前客户端合同兼容。收紧权限前先确认管理员仍有可用成员文档；开放删除权限前先在 Emulator 运行查看者、管理员和插件用户矩阵。生产数据变更前导出受影响日期范围。

## 插件

ImportTool 回滚只替换插件代码和发布配置。`enabled=false` 是保留的紧急可用性旁路：关闭后插件不加载认证、网络、队列或上传模块，不能把该旁路误当成数据撤销。重新启用后，原 Firebase UID 队列继续隔离上传。

## 记录

每次回滚记录版本、合同版本、操作者、原因、影响范围、验证账号和恢复条件。至少复核两套用途边界、事件去重、脱敏、网页角色和插件关闭旁路。
